## 주요 결정 사항

| 항목 | 결정 | 이유 |
|------|------|------|
| 임베딩 모델 | `gemini-embedding-2` | `text-embedding-004` 접근 불가 확인 |
| 벡터 차원 | 3072 | `gemini-embedding-2` 기본 차원 |
| 벡터 인덱스 | 미생성 | Supabase pgvector 2000차원 제한으로 추후 추가 |
| 청킹 전략 | `\n` 단위 분리, 최소 10자 | 짧은 줄바꿈 문서 대응 및 총 14개 청크 생성 확인 |
| 중복 업로드 | 기존 데이터 덮어쓰기 처리 | 동일 파일 재처리 편의성 고려 |
| API 한도 이슈 | 무료 플랜 일일 한도 있음 (자정 리셋) | 429 에러 방지용 딜레이(2초) 추가 및 예외 처리 |
| 답변 생성 모델 | `gemini-2.5-flash` | 빠른 속도와 성능을 고려하여 최종 확정 |
| 임베딩 최적화 | `gemini-embedding-2` 배치 처리 | API 호출 횟수 최적화 (14번 → 1번) |
| 렌더링 라이브러리 | `react-markdown` | SOP, 체크리스트, CS 답변 결과를 UI에 마크다운으로 렌더링 |
| CS 답변 정책 | 할루시네이션 방지 | 문서에 없는 내용은 지어내지 않고 담당 부서 안내로 처리 |
| Toast 알림 | 전역 상태 관리 | `ToastContext`로 전역 관리, 3초 자동 소멸 처리 |
| Empty State | 상태별 적용 | 업로드 탭, 검색 탭 각각 상황에 맞는 안내 적용 |
| 마크다운 렌더링 | 일관성 확보 | SearchPage도 `react-markdown` 적용 완료 |
| 복사 버튼 | UX 개선 | 클릭 시 "복사됨" 상태 변경 및 Toast 알림 연동 |
| 내일 할 것 | 포트폴리오 퀄리티 업그레이드 | UX 개선 및 테스트 진행 |
| CS 탭 | 사내 문의 답변으로 변경 | 외부 고객 CS에서 내부 운영 지원 목적성 강화 |
| 데모 문서 | 5종 샘플 문서 작성 | 업무매뉴얼, 신입온보딩, 경비처리, 회의실예약, 워크샵규정 준비 완료 |
| README | 구성 요소 완성 | 문제정의/기능/아키텍처/기술선택이유/트러블슈팅/회고 작성 완료 |
